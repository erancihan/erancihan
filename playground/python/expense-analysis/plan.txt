# Statement Period & Card Filtering Update

## Goal
Transform the main Expenses view into a "Statement Month" view that respects a configurable cutoff date (e.g. the 28th of the month) instead of raw calendar months, and introduce per-card filtering tabs.

## Backend Changes (`src/web.py`)
1. **Date Filtering Update:** Update the `from` and `to` parsing in `api_expenses` and `api_summary_monthly` to support exact dates (`%Y-%m-%d`) in addition to `%Y-%m`. This allows the frontend to request exact statement periods (e.g., `2026-03-29` to `2026-04-28`).
2. **Card Filtering:** Add a new `card` query parameter to `api_expenses` and apply `query.filter_by(card_number=card)` if provided.

## Frontend Changes (`static/index.html`)
1. **State Modifications:**
   - Add `statementCutoffDay: 28` (saved to localStorage so it persists).
   - Replace `dateFrom` and `dateTo` with a single `statementMonth` selector (e.g., `2026-04`).
   - Add `selectedCard: ''` state to track the active tab.
   - Add a computed method to calculate the exact `from_date` and `to_date` based on the `statementMonth` and `statementCutoffDay`.
2. **UI Updates:**
   - Add an input field/settings gear for "Cutoff Day" (default: 28).
   - Change the Date filters above the table to a single "Statement Month" dropdown/input.
   - Add a row of Tabs above the table: `[All Cards] [****1234] [****5678]` populated from the existing `cards` array.
   - The selected tab will update `selectedCard` and trigger a re-fetch of the expenses table.

## User Review Required
> [!IMPORTANT]
> The summary charts at the top of the dashboard currently group data by raw calendar month (`%Y-%m`). Should these charts also align with the custom statement period (e.g., showing the statement period total instead of the strict calendar month total)?
